Findings that shape the plan

The Aurora shell is already in main

Layout v10 ships TopBar + SubNav + ⌘K palette + a 5-tab mobile bar (Home / Track / Plan / Analyze / Profile) + Add FAB (tap = add txn, long-press = speed dial) + an Ask Vyact drawer. The revamp keeps this navigation model and redesigns what's inside it — journeys stay familiar, every screen gets the full Aurora treatment.

No Goals module in production

Goals exist only as data feeding the insights engine — “Goals are not a module.” The earlier mobile mockups' Goals tab won't be carried over; the real Plan section (Budgets · Debts · Net Worth · Accounts) takes its place.

Modules flex by household type

Templates gate navigation: Single hides Members/Splits, Family shows all, Self-employed hides Splits and focuses Net Worth + Debts. Designs will show the Family (fullest) set.

Estimates are a first-class concept

Onboarding seeds an estimated baseline that “converges to confirmed” over 21 days — EstimatedTag, confidence badges, “Picture confirmed %”. The revamp gives estimated vs confirmed a clear visual voice (dashed vs solid, honey tint).

Mobile navigation — the 5-tab map

Adopting the section model already shipped in main, refined. Every one of the 16 routes stays reachable: primary via tabs, secondary via the pill subnav inside each section, everything via ⌘K / search. The FABs survive: + (Add) and ✦ Ask Vyact.

Forms — the “half-screen, fewest taps” doctrine

Your stated pain point. Every add/edit flow is rebuilt on one pattern: a bottom half-sheet (mobile) / compact glass dialog (desktop) where tappable chips replace open boxes and dropdowns, smart defaults pre-fill everything, and the primary case commits in ≤3 taps.

The pattern, using Add Transaction
  • Amount first. Big mono amount field, auto-focused numeric pad. Type (Expense / Income / Transfer / Invest) is a segmented chip row — defaults to Expense.
  • Category = emoji chip grid, your 8 most-used first (“recents learn you”), search for the rest. No dropdown.
  • Date & account are chips: Today · Yesterday · 📅, and account chips with the default pre-selected. Description is optional, auto-suggested from category.
  • Everything else folds away under “More” — note, member, currency, split, mark-recurring. The 90% case never sees it.
  • Commit & celebrate: one big Save, toast + tick; “Save & add another” for batch entry.

Same doctrine applied to Add Budget (month chip + total + allocation chips w/ suggested amounts), Add Debt, Add Asset/Account (type-tile first, then 2–3 fields), and Recurring — which becomes a sentence builder: “Repeat monthly on the 1st · auto-approve · remind 3 days before.”
