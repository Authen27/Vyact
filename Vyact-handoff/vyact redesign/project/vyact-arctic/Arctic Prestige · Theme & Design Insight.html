A cool ice-and-steel re-skin of the Vyact operator console, delivered as a two-layer token system — fixed brand palette plus semantic tokens that flip between light and dark. Drop arctic-tokens.css into the design system, scope it to the admin app, and build only on the semantic names so both modes track for free.

05 Usage rules Keep the system honest across both themes.

Do

  • Reference semantic tokens (--arc-surface, --arc-ink-2) so light and dark resolve automatically.
  • Use status tints for fills and the solid status color for text/icons — the pairing is contrast-checked in both modes.
  • Lead with the glacier accent for brand moments; keep it to ~5% of the surface.
  • Set data-theme once on <html> before paint to avoid a flash.

Don't

  • Hard-code raw hexes in components — they won't flip with the theme.
  • Reuse the dark accent (icy #C7E2F2) on a light surface; it fails contrast.
  • Introduce new hues outside the six-swatch palette for admin chrome.
  • Mix warm FinFlow tokens (--ff-*) into the admin surface — Arctic is a full replacement here.
06 Extend it Scope to admin, switch themes, add a token. 
/* 1 · load + scope to the admin app only */
<link rel="stylesheet" href="arctic-tokens.css">
<html data-theme="dark">     /* or "light" */

/* 2 · flip at runtime + persist */
root.setAttribute('data-theme', next);
localStorage.setItem('vyact-admin-theme', next);

/* 3 · add a token — define it in BOTH blocks */
[data-theme="dark"]  { --arc-accent-2: #7FA8D4; }
[data-theme="light"] { --arc-accent-2: #3F6FA0; }

Two layers, one rule. The raw palette is the brand's fixed vocabulary; semantic tokens are the only thing components touch. To theme a new surface, you never invent colors — you remap existing semantic tokens.

Admin-only by design. These tokens use the --arc-* namespace and the .arc scope class, so they sit alongside the warm --ff-* consumer tokens without collision. Apply .arc + data-theme on the admin root only.

Adding a token? Always define it in both theme blocks. A token that exists in only one mode is the most common source of theme drift.
